# Per-Sample Coordinate Record Schema

Per-sample records are stored as CSV or gzipped CSV tables. The canonical
`l2` record is `records/canonical_l2/per_sample_coordinates.csv.gz`.

## Core Identifier Fields

- `encoder`: Encoder identifier.
- `pretty_encoder`: Display name when available.
- `family`: Architecture or pretraining family.
- `sample_id`: Validation sample identifier in the ImageNet-100 audit split.
- `class_id`: True class identifier in the audit split.
- `clean_correct`: Whether the frozen encoder plus linear head predicts the
  true class at the clean input.
- `fixed_competitor_k_star`: Clean nearest competitor used by the fixed-pair
  margin.

## Margin Fields

- `M`: Clean fixed-competitor margin.
- `Lp`: First-order attack scale. For `l2`, this is `eps * ||g||_2`. For
  `linf`, this is `eps * ||g||_1`.
- `D_FGSM`: One-step margin drop. Budget-suffixed aliases such as
  `D_FGSM_10` are included when generated by a budget-specific pipeline.
- `D_PGD_10`, `D_PGD_50`, `D_PGD_100`, `D_PGD_200`, `D_PGD_500`: Iterative
  margin drops under the stated PGD budget when available.

## Coordinate Fields

- `lambda`: Margin slack ratio `M / Lp`.
- `kappa`: On-ray shortfall `(Lp - D_FGSM) / Lp`.
- `rho_10`, `rho_50`, `rho_100`, `rho_200`, `rho_500`: Iterative-over-one-step
  margin gain normalized by `Lp`.
- `tau_10`, `tau_50`, `tau_100`, `tau_200`, `tau_500`: Net residual
  `rho_k - kappa`.
- `m_10`, `m_50`, `m_100`, `m_200`, `m_500`: Survival margin
  `lambda + kappa - 1 - rho_k`.
- `pass_10`, `pass_50`, `pass_100`, `pass_200`, `pass_500`: Fixed-competitor
  robust-retention indicator for each available budget.

## Notes

Missing values indicate that a budget was not run for that encoder or
protocol. All coordinate identities are evaluated on clean-correct samples
with `Lp > 0`.
